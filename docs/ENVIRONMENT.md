# Environment Setup

Цей документ описує налаштування середовища для розробки та деплою проєкту.

## Локальна розробка

### Встановлення

```bash
npm install
```

### Запуск розробницького сервера

```bash
npm run dev
```

### Доступ

Відкрий `http://localhost:5173` у браузері.

## Змінні оточення

Проєкт використовує Supabase для автентифікації та збереження даних. У локальному середовищі потрібно додати файл `.env` або `.env.local` з такими змінними:

```env
VITE_SUPABASE_URL=https://your-supabase-project-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
```

### Пояснення

- `VITE_SUPABASE_URL` — URL вашого Supabase проекту.
- `VITE_SUPABASE_ANON_KEY` — публічний анонімний ключ для клієнта.

> Зберігай секрети поза репозиторієм. Не додавай `.env*` до git.

## Деплой на Vercel

На Vercel додай ті самі змінні оточення у налаштуваннях проекту.

### Рекомендувані параметри

- Build command: `npm run build`
- Output directory: `dist`

## Важливі файли

- `vite.config.ts` — конфігурація Vite
- `package.json` — скрипти та залежності
- `tsconfig.json` — налаштування TypeScript
- `eslint.config.js` — правила ESLint

## Безпека

- Не зберігай приватні ключі у git.
- Використовуй `VITE_` префікс тільки для тих змінних, які потрібні у фронтенді.
- Якщо додаєш бекенд-секрети, зберігай їх у серверних змінних оточення.
