# Architecture

Цей документ описує загальну архітектуру проекту.

## Структура `src`

- `App.tsx` — кореневий компонент додатку.
- `main.tsx` — точка входу, яка підключає React і Vite.
- `routes/AppRoutes.tsx` — конфігурація маршрутів.
- `components/` — повторно використовувані UI-компоненти.
- `pages/` — окремі сторінки додатку.
- `data/` — статичні дані та контент.
- `lib/` — об’єкти конфігурації, наприклад Supabase.
- `hooks/` — кастомні React hooks.
- `seo/` — логіка метаданих сторінок.

## Маршрути

Маршрути зберігаються у `src/routes/AppRoutes.tsx`.

- Використовується `react-router-dom`.
- Більші сторінки імпортуються динамічно через `lazy()`.
- Всі маршрути обгорнуті в `Suspense`, що показує загрузку під час завантаження сторінок.

## Основні компоненти

- `Header` — навігація та логотип.
- `Footer` — нижній колонтитул.
- `Assistant` — допоміжний інтерфейс.
- `WelcomePopup` — привітальне вікно.
- `CookieConsent` — банер з cookie consent.
- `UpdatePrompt` — сповіщення про оновлення.
- `AssistantPromo` — промо для спільнот.

## Auth / Supabase

- Логіка авторизації реалізована в `src/lib/supabase.ts`.
- У `App.tsx` зчитується поточна сесія через `supabase.auth.getSession()`.
- При зміні стану авторизації використовується `supabase.auth.onAuthStateChange()`.
- В деяких маршрутах доступ обмежено: якщо немає сесії, показується компонент `Auth`.

## PWA

- Налаштування в `vite.config.ts` через `vite-plugin-pwa`.
- У manifest вказані `name`, `short_name`, `theme_color`, `background_color`, `icons`.
- Зазначені runtimeCaching правила для кешування Google Fonts та іншого трафіку.
- `registerType: 'autoUpdate'` оновлює сервіс-воркер автоматично.

## Загальні принципи

- Пріоритет на просту архітектуру компонентів.
- Використання TypeScript для типізації.
- Lazy-loading сторінок для кращої продуктивності.
- Окрема документація для змісту та налаштувань.
